MikeDEV


Seems like we're having a bit of conflict. Let me clarify. This is Meower's 100% honest clarification.

Meower and Yoom are 2 sides of the same coin, they will share the same fundamental code. Meower will be restricted in its features since we have to comply with the rules. Yoom will serve as a playground for these features.

Here are the differences:

Meower (Scratch, Scratch hosted)
> Uses Cloud Variables
> Relies upon p2p (Peer-2-Peer) comms for file storage
> Custom PFP functionality disabled
> Predefined phrase-based chat*
> Search functionality limited
> No user quotes on profile pages
> Pseudo Real-Time chats
> Private comms are not possible, all comms are public

Yoom (Compiled HTML, independently hosted)
> CloudLink-based
> Uses custom cloudlink-based server for file storage
> Custom PFP support + User camera support (will require consent to privacy policy)
> Traditional, hyper-sensitive whitelist-based chat (+ support for a phrase-based chat if needed)
> Full-text search
> True real-time chat
> Direct Messages support
> Experimental “calling” support (will be using a hyper-sensitive whitelist)

Both:
> Profile pages will have basic user status (online, offline, idle, away, do not disturb, banned/kicked)
> Rely upon servers for storage/comms
> Allow moderated, secure chat
> Block sharing of personal info (emails, addresses, phone numbers, etc.)
> Have easter eggs
> Have a sense of humor
> Emoji support (Due to technical complications, Emojis will be added in a later update)

*The GUI will allow you to select and assemble sentences using pre-approved phrases. HOWEVER, they will only be permitted if they are within the permitted predefined phrases list… The phrase assembler is a more user-friendly approach to this phrase-only chat limitation.

The only reason why the preview allows free text is that the phrase selector is not complete. Free text support will be removed in the release candidate of Meower.

I hope I cleared up some things. I'm not lying to anyone. If you feel like there is something wrong with Meower, you can remix it and fix issues, and request a review of the code for merging. Meower is FOSS, and we should embrace it.
