legality argument incoming

To be honest, we shouldn't really care about the licensing. The license will technically be invalid for the project released onto Scratch, however, the license applies to the original copy(ies) of the code + all of the (applicable) sister projects (so far, I'm making Yoom unlicensed)

TL;DR version of Unlicense Vs. Scratch:

Unlicense
  • Public domain (free to everyone and anyone without restrictions)
  • Crediting the original authors isn't required (though would be nice)
  • Using the same license isn't required in redistributions
Scratch (CC BY-SA 2.0)
  • Credit is required for all redistributions of the code (“Thanks to X for Y…”)
  • The license must remain the same in distributions
Both
  • Anyone can view/modify/compile/distribute the code (“View inside” / “Remix”)
  • Credits are applicable
  • Denies copyright

In short, they're pretty compatible…?

Edit: nvm, however we will need to mention the licensing disclaimer in the legal section of the thread

the final release of the project code (that is on Scratch) will be and shall be using CC BY-SA 2.0, BUT any copy of the project code that is NOT on Scratch is subject to Unlicense.
