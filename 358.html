…Which is why I'm improving the security of the services that Meower will rely upon. I would like to keep Meower secure by design.

Since Meower will depend on CloudLink Suite, CloudLink itself will need to depend upon the security of both CloudAccount and CloudDisk. My biggest improvement from old CloudLink is the implementation of 2-Factor Authentication, as well as OTSKs (One-Time Session Keys), which are special token keys that can only be used for one logged in user, and can be used to authenticate other services (which in this case, CloudDisk).

My biggest concern regarding security is obvious stupidity on my part. One obvious security note is to keep the Meower server IP private, and purged from source code, and only stored (with very good obfuscation) in compiled binaries/HTML files. I'm also considering implementing a new security feature for CloudLink servers, and that is Trusted Access keys. these keys can be generated on the server-side and can be stored in a project as a variable. Ideally, you would compile the project and have the project send that code to the server to verify access, and purge the variable's data when distributing open-source code. Think of it as something similar to SSL.
