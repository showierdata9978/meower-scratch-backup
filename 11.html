In defense of the argument, That's fundamentally what this generator is. It's fundamentally a whitelisted phrase generator, a more elegant system that I'll see if we can implement (if not, we'll have to generate a bulk list of all allowed phrases… which isn't elegant). (Please see the comments in the project for more info.)

Fundamentally, the generator ultimately is just a selector for different combinations of whitelisted phrases. For example…

“Hello”, “Hi”, “Welcome”, and “Bye” are allowed words that can be also whitelisted phrases. Forming unique combinations of these also form whitelisted phrases. “Hello! Welcome!” would be allowed, while something more random (as if it were typed out manually): “hello hello hello hello” wouldn't be permitted as it's not in the whitelist.

Implementing a phrase generator that creates permitted phrases would still technically comply with the rules, but again I'm still not 100% sure at this time. I really don't want to have users select an entire sentence from a long, boring list of prewritten phrases when you can assemble a phrase in the generator and still have it pass through a whitelisted phrase verification check. The design is simple enough to use, and does work, as this is fundamentally what Disney did back when Club Penguin was still around (and assuming your account was set to safe chat mode).

If It's not allowed, I don't mind complying with the rules. But I would really like a second opinion from the ST before we make a final decision.
