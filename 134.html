MikeDEV wrote:

Also, I should provide a little bit of documentation on how the phrase system works.

There is a list called “WORDS”, which contains the first component of the phrase system. This list consists of single words, which should be used to create phrases.

Each word is assigned to an item number in the list, and these item numbers are used for loading categories and verifying phrases.

For example:

1. Hello
2. World
3. !

In category x, it will contain the following:

1;2;

and in category y, it will only contain:

3;

the “CATEGORY_CONTENTS” list is already prepared for the GUI. The selector for the categories are in order with the contents list, and the 7th item in that list is for the “? ! .” button.

And finally, the “PHRASE_DB” list contains all the permitted combinations. This should take into consideration the above. Also, using the EZ-Make Button API, can someone please program in a backspace function?
