As far as the CloudLink-based fork of meower goes, I'm using server-side SHA-256 hashing for my database's password data. This is really bare-bones, but it does help.

DabDatBass wrote:

I just realized this, but we need a way to make a new line. “\n” makes sence.

Also, I need a project that has both the pen and zed gui in it and the cloud engine.

My preview GUI's BBCode+ engine supports this, through the use of the newline tag: 
[n]

You don't have to use
[/n]
to end the newline. It will be able to read end newline tags, but it won't do anything. Only the start newline tag draws the next text data under a new line.

For example, someone could create a post (using the phrase selector, of course) and stylize it. They can declare a new line as simple as this:

[b]Hello! This is text that's not on a new line.[/b][n]Here is some text that is on a new line. ([/n] optional)

Also just an FYI, the actual fancy boxes and buttons (not present ATM) on posts are contained within the 
[container] and [/container]
tags.

[container=Test]This is a sample container.[/container]

If someone needs to create an empty container (or for a GUI element), use a space (this is currently a bug that I will not be fixing until the release canidate is published and live.)

[container= ]Oops, an error occured.[/container]
