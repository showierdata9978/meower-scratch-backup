Glitch report I guess
I found something that I am pretty sure is a bug. After clicking the user icon the context menu (or however it's called) opens, The problem is that buttons that are under menu can be clicked through the menu. Meower quite well masks this problem because the things that are happening after clicking on context menu buttons are overlapping the buttons under menu functions, because they transfer you to like other scene. The only way you can see this working is when clicking on context menu background when under it is a button from underneath, but then again Meower masks it well because it hides the context menu when clicked on its background and not on buttons, so it looks more properly. This bug may be annoying when someone misses clicking on My Profile button, clicks on context menu empty orange background and accidentally hides the post because the hide post button was underneath the context menu in the place where he clicked.

Maybe there should be a layer system in the PIRE, every object would have a selected layer and then for example button click wouldn't work if at the same time the mouse would overlap the rectangle that is on the layer that is above the layer on which the button object is.
