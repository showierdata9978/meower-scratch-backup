import string
words = ["@(ausername)", "?", "!", "."]
sentences = ["@(ausername)"]

"""

Meower PhraseDB Converter v1.1
0BSD License

Required file: "INPUT.txt" - Contains a list of human-readable phrases
Optional file: "REACTIONS.txt" - Contains a list of ASCII emotes

Outputs:
PHRASE_DB.txt - Load contents into "PHRASE_DB" list in Meower
WORDS.txt - Load contents into "WORDS" list in Meower
PHRASE_READBACK - Debugging file, reconverted output of PhraseDB

"""

def saveList(name, data):
    wordfile = open(name, "w")
    for element in data:
        wordfile.write(str(element))
        wordfile.write("\n")
    wordfile.close()

try:
    ## Step 1: Split words while removing punct at the end of the lines, save
    with open("INPUT.txt") as f:
        lines = f.read().splitlines()
    for x in lines:
        inp = x.split()
        out = []
        for y in inp:
            charout = y.replace("!", "")
            charout = charout.replace("?", "")
            charout = charout.replace(".", "")
            out.append(charout)
            if not charout in words:
                if not len(charout) == 0:
                    words.append(charout)
        out2 = ""
        for z in out:
            out2 = out2 + z + " "
        sentences.append(out2)

    ## Step 2: Convert output of words into barebones PhraseDB
    b = []
    for f in words:
        b.append(f)
    phrases = []
    for x in sentences:
        y = x.split()
        z = ""
        for a in y:
            z = z + str(b.index(a)+1) + ";"
        if not z in phrases:
            phrases.append(z)
     
    ## Step 3: Check for similar phrases (mainly for punctuation) and add to PhraseDB
    with open("INPUT.txt") as f:
        lines = f.read().splitlines()
    for phrase in phrases:
        if not len(phrase) == 0:
            checkPhrase = ""
            for chk in range(len(phrase.split(";")) - 1):
                checkPhrase = checkPhrase + words[int(phrase.split(";")[chk]) - 1] + " "
            checkPhrase = checkPhrase[:len(checkPhrase)-1]
            for test in ["?", "!", "."]:
                #print(checkPhrase + test)
                for line in lines:
                    if str(line) == (checkPhrase + test):
                        phrases.append(phrase + str(["?", "!", "."].index(test) + 2) +";")

    ## Step 4: Add reactions
    try:
        with open("REACTIONS.txt") as f:
            lines = f.read().splitlines()
        for out in lines:
            sentences.append(out)
            words.append(out)
        for x in sentences:
            y = x.split()
            z = ""
            for a in y:
                z = z + str(words.index(a)+1) + ";"
            if not z in phrases:
                phrases.append(z)
    except FileNotFoundError:
        pass

    ## Step 5: Write PhraseDB to disk
    saveList("PHRASE_DB.txt", phrases)
    saveList("WORDS.txt", words)

    ## Step 6: Read PhraseDB back into human-readable format (Debugging use only?)
    with open("PHRASE_DB.txt") as f:
        lines = f.read().splitlines()
    readback = []
    for item in lines:
        conv = item.split(";")
        del conv[len(conv)-1]
        out = ""
        for i in conv:
            out = out + words[int(i)-1] + " "
        out = '"' + out + '"'
        readback.append(out)
    saveList("PHRASE_READBACK.txt", readback)
except FileNotFoundError:
    print("Please put a file named INPUT.txt into the same directory as this script!")

Also, I should provide a little bit of spec on how we will be encoding/decoding messages.

When the user selects a phrase in the GUI, if the contents of the created phrase exist as an item within the phrase database, it will be allowed to be sent, and the phrase ID is assigned to a number in the phrase database.

We string together several of these IDs to create larger conversations. This effectively conserves bandwidth since we don't need to transmit individual chunks of a phrase, but rather, an association of said phrase.

As for items that contain the @(username) item, the very next item in the message will encode the username to @t. Here's an example of how this should work in the end:

WORDS list:

1. “Hello”
2. “World”
3. “!”
4. “.”
5. “@(username)”

PHRASE DB list:

1. “1;2;3;”
2. “1;2;4;”
3. “1;5;3;”

When the user inputs a phrase, they select words from the word list. Each item in the word list has an item number, which tells the PhraseDB what to reference. If a certain combination of words doesn't match any of the contents of the phrase DB, the item will not be added to the message or the message will be prevented from being transmitted.

Let's say that a user inputs “Hello”, “World”, and “!”, creating the phrase “Hello World!”. This phrase is encoded as “1;2;3;”. Since this exists within the phrase DB, it is allowed. And this item, existing within the DB, is item number 1.

When the user transmits the message, the full message is made as simple as just “1;”, representing the item number 1 in the DB, which will then decode into item “1;2;3” in the words list, which also gets decoded into “Hello World!”.

The trick here is that if a certain combination of words isn't within the predefined phrase database, the message will never be sent, and in the event that the message does send, the data would be corrupted and the users on the other end would never see the message, as it would automatically be detected as corrupt and deleted upon receiving.

Now, phrases that utilize the username tagger will simply add the username to @t right after the PhraseDB item number. Let's go over it.

1. User inputs “Hello”, “@(username)”, and “!”.

2. These words are encoded as “1;5;3;”, and this item exists within the phrase DB, which allows the message. This phrase is encoded as Phrase ID #3.

3. The user is asked what username to @t. They will only be permitted to select from an automatically updating list of online users. Let's say that the user selects “another_user”.

4. The user has essentially created the message “Hello another_user!”. This is ultimately encoded as “3;another_user”, which gets sent to everyone, but the message will mention the user, “another_user”.

5. Let's say that another_user gets the message. The project decodes the message “3;another_user” as Phrase #3, with the username tag set to “another_user”.

6. Phrase 3 is decoded as words 1, 5, and 3. Since the message specifies the value of the @(username) tag (word 5), that will be replaced with the value “another_username”.

7. The message “Hello another_username!” is verified and shown to another_user.
